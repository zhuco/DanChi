# FSRS 调度与入口说明

## 入口原则

当前版本只有一个主学习入口：**开始提分**。

用户不需要在“新学”“复习”“牢记模式”之间选择。点击“开始提分”后，系统自动生成今日学习队列，并把新词、短期学习、遗忘重学和长期复习合并到同一条 FSRS 队列中。

首页的“拼写”“选义”“听写”属于自由专项练习，用于额外训练，不作为主学习入口。

## 队列优先级

每日进入学习页时，按以下顺序生成任务：

1. `learning` / `relearning` 且 `due_at <= now` 的卡片。
2. `review` 且 `due_at <= now` 的卡片。
3. 还没有 recognition card 的新词。

所有复习都以 `card.due_at` 为准，不再使用“复习前 N 天单词”或固定艾宾浩斯天数表。

## 新词数量

新词数量动态计算：

- 默认每日学习时长：10 分钟。
- 统计最近 200 次答题平均耗时。
- 如果到期卡片预计耗时已经超过每日时长，当天不补新词。
- 如果还有剩余时间，再补入新词。
- 新词每日上限默认 20 个，可在学习计划中调整。

## 卡片模型

不要把 `word` 当成唯一记忆对象。一个单词可以对应多张卡片。

当前第一版实现：

- `recognition`：英文单词 -> 4 选 1 中文释义。

后续预留：

- `recall`：中文释义 -> 拼写英文。
- `audio`：听发音 -> 选择释义。
- `sentence`：例句填空或语境选择。

## 答题流程

新词第一次出现时先测试：

1. 展示英文单词和 4 个中文释义选项。
2. 用户作答。
3. 根据正确性、耗时、是否使用提示映射 FSRS rating。
4. 调用开源 `java-fsrs` 生成新的卡片状态和下次到期时间。
5. 写入 `review_logs`。
6. 展示单词详情页，包括词性、释义、例句、助记和发音。

信息页本身不算一次复习，不写入 `review_logs`。

## Rating 映射

FSRS 使用四档 rating：

- `Again = 1`
- `Hard = 2`
- `Good = 3`
- `Easy = 4`

recognition 选择题映射规则：

- 首答错误：`Again`
- 首答正确但使用提示：`Hard`
- 首答正确且耗时超过 8000ms：`Hard`
- 首答正确且耗时 3000ms 到 8000ms：`Good`
- 首答正确且耗时不超过 3000ms：
  - 新卡第一次学习最高映射到 `Good`
  - review 阶段且最近连续正确才允许 `Easy`
  - 其他情况映射到 `Good`

## 数据记录

核心表：

- `words`：词条内容、词性、释义、例句、词库归属。
- `cards`：FSRS 记忆对象，保存 `state`、`dueAt`、`stability`、`difficulty`、`reps`、`lapses`。
- `review_logs`：每次答题记录，保存 rating、耗时、题型、选项数、首答是否正确、是否提示、答题快照、前后状态。
- `user_fsrs_setting`：用户 FSRS 设置，保存目标记忆率、最大间隔、短期学习步骤和参数 JSON。

## 掌握度展示

不使用永久“已掌握”作为主调度状态。展示层可从 FSRS 状态动态推导：

- `due_at <= now`：待复习。
- `lapses >= 2` 或 `difficulty >= 8`：薄弱。
- `stability >= 30` 且当前 retrievability 达到目标记忆率：较稳。
- `stability >= 90`：长期掌握。

## 扩展方向

新增 recall、audio、sentence 时，不新增主入口。做法是：

1. 为同一个 `word_id` 创建新的 `card_type`。
2. 使用对应题型采集答题结果。
3. 映射到 `Again/Hard/Good/Easy`。
4. 继续调用同一个 FSRS 调度模块更新 `cards`。
5. 继续写入统一 `review_logs`。
